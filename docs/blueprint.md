# 프로젝트 청사진

## 서비스 한 줄 설명

Campaign Hub는 블로그, 인스타그램, 유튜브 등 여러 플랫폼의 체험단 모집 정보를 모아 사용자가 검색, 필터, 정렬로 원하는 체험단을 빠르게 찾는 웹 서비스입니다.

## 문제 정의

체험단 모집 정보는 여러 사이트에 흩어져 있고, 사이트마다 검색 방식과 카테고리 구성이 다릅니다. 사용자는 원하는 지역, 키워드, 플랫폼, 마감일 조건에 맞는 체험단을 찾기 위해 여러 사이트를 반복해서 확인해야 합니다.

## 해결 방향

- 체험단 정보를 표준화된 데이터 구조로 저장
- 제목, 업체명, 키워드, 지역, 플랫폼, 카테고리 기준으로 탐색 제공
- 마감일과 인기도 기준으로 중요한 체험단을 먼저 노출
- 회원별 찜 데이터를 기반으로 추천과 알림으로 확장
- 초기에는 관리자 직접 등록, 이후 크롤링 배치로 자동 수집

## MVP 범위

MVP는 비회원도 사용할 수 있는 조회 중심 서비스로 시작합니다.

- 체험단 목록 조회
- 최신순, 마감순 정렬
- 체험단 상세보기
- 제목, 업체명, 키워드 검색
- 지역, 플랫폼, 카테고리 필터
- 관리자 수동 등록

## 확장 범위

- 회원가입, 로그인
- 즐겨찾기
- 마이페이지
- 알림
- 지도 검색
- AI 자연어 검색
- 추천 시스템
- 외부 체험단 사이트 크롤링

## 주요 화면

| 화면 | 설명 |
| --- | --- |
| 홈/목록 | 검색창, 필터, 정렬, 체험단 카드 목록 |
| 상세 | 제공 내역, 모집 기간, 모집 인원, 신청 링크, 찜 버튼 |

| 관리자 등록/수정 | 체험단 정보 입력 및 수정 |

## 핵심 도메인

### Campaign

체험단 모집 정보입니다. 제목, 업체명, 제공 내역, 모집 기간, 모집 인원, 신청 링크, 지역, 플랫폼, 카테고리, 조회수, 인기도 점수를 가집니다.

### Member

서비스 회원입니다. 로그인 정보, 권한, 관심 카테고리, 관심 지역 등을 가집니다.

### Favorite

회원이 찜한 체험단입니다. 추천과 마이페이지의 기반 데이터가 됩니다.

### Category

맛집, 뷰티, 운동, 숙박, 여행 등 체험단의 주제 분류입니다.

### Platform

블로그, 인스타그램, 유튜브 등 체험단 참여 채널입니다.

### CampaignSource

체험단 정보의 출처입니다. 직접 등록인지, 특정 외부 사이트에서 수집했는지를 관리합니다.

## 1차 구축 구조

현재 1차 구축은 공통 도메인과 사용자/관리자 기능을 분리한 코로케이션 구조입니다.

```text
com/campaignhub/
  campaign/
    domain/
    repository/
  user/
    campaign/
      controller/
      service/
      dto/
  admin/
    campaign/
      controller/
      service/
      dto/
```

사용자는 모집 중인 체험단을 조회하고, 관리자는 전체 체험단을 등록, 조회, 수정, 삭제합니다.

크롤링 확장 후:

```text
Scheduler / Batch
  |
Crawler
  |
Normalizer
  |
Campaign Service
  |
Database
```

## 검색 전략

### 1단계

- SQL `LIKE` 검색
- 제목, 업체명, 키워드 컬럼 대상
- 지역, 플랫폼, 카테고리는 정확히 일치하는 필터

### 2단계

- QueryDSL 또는 Specification으로 동적 검색 조건 관리
- 복합 조건 검색 최적화

### 3단계

- Elasticsearch, OpenSearch, Meilisearch 중 하나로 검색 엔진 확장
- 형태소 분석 기반 검색 품질 개선

## AI 검색 전략

사용자가 `"서울 강서구 맛집"`처럼 자연어로 입력하면 다음 구조로 분석합니다.

```json
{
  "region": "서울",
  "district": "강서구",
  "keyword": "맛집",
  "category": "맛집"
}
```

초기에는 룰 기반 파싱으로 구현하고, 이후 LLM API를 붙여 자연어 분석 정확도를 높입니다.

## 추천 전략

초기 추천은 단순 점수 기반으로 시작합니다.

- 사용자가 많이 조회한 카테고리
- 사용자가 많이 찜한 카테고리
- 사용자의 관심 지역
- 마감이 임박한 체험단
- 조회수와 찜 수가 높은 체험단

추천 점수 예시:

```text
score = categoryMatch * 3
      + regionMatch * 2
      + favoriteCount * 1.5
      + viewCount * 0.1
      + deadlineBoost
```
